# 🤖 Agenten-Anweisung: TODO-Tracking

## Zweck
Diese Anweisung beschreibt, wie du als Agent die `TODO.md` Datei korrekt befüllst und aktualisierst.

---

## 📋 Aufgabe

Wenn du während deiner Arbeit Probleme, Warnungen oder Verbesserungsmöglichkeiten identifizierst, dokumentiere diese in der `TODO.md` Datei.

---

## 🎯 Kategorisierung

Ordne jeden Fund einer der folgenden Kategorien zu:

| Symbol | Kategorie      | Wann verwenden?                                                |
|--------|----------------|----------------------------------------------------------------|
| 🔴     | **KRITISCH**   | Sicherheitslücken, Bugs die Produktion blockieren, Datenverlust |
| 🟠     | **WARNUNG**    | Performance-Probleme, veraltete Dependencies, potenzielle Bugs  |
| 🟡     | **VERBESSERUNG** | Code-Qualität, Refactoring, Dokumentation, Nice-to-haves       |
